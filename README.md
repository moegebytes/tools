# Hime Tools

Rust CLI tools for translating games using the NVS/FVP visual novel engine.

Supports packed archives (`.bin`), NVSG images, and HCB bytecode scripts.

## Disclaimer

This project was born as an experiment to learn and get familiar with current LLMs and their capabilities. Everything was written from scratch using Claude Code. A human in the loop has read code and directed LLM to specific architectural choices. In rare cases, where LLM failed, manual code changes were implemented. Care was taken to avoid unnecessary dependencies and to keep transitive dependencies to minimum where possible. Choice of using older versions of certain Rust libraries was explicit. Following content was generated by LLM and adjusted later on.

## Installation

```
cargo build --release
```

The binary is at `target/release/hime-tools` (or `hime-tools.exe` on Windows).

## Usage

### Archives

List files in a `.bin` archive:
```
hime-tools bin ls archive.bin
```

Unpack all files from a `.bin` archive:
```
hime-tools bin unpack archive.bin output/
```

Extract a single file by name:
```
hime-tools bin get archive.bin filename [output]
```

Pack a folder back into a `.bin` archive:
```
hime-tools bin pack input/ output.bin
```

The pack command sorts entries case-insensitively to satisfy the engine's binary search. Filenames are encoded as Shift-JIS.

Replace a single file in an archive without full unpack/repack:
```
hime-tools bin replace archive.bin filename replacement
```

Validate archive structure, data regions, and sort order:
```
hime-tools bin validate archive.bin
```

### Images

Inspect NVSG image metadata without decompressing pixel data:
```
hime-tools nvsg info input.nvsg
```

Decode an NVSG/HZC1 image to PNG:
```
hime-tools nvsg decode input.nvsg [output.png]
```

If `output` is omitted, it defaults to the input path with a `.png` extension.

On decode, a re-encode hint is printed to stderr with all the header values you need to pass back.

Encode a PNG to NVSG/HZC1:
```
hime-tools nvsg encode -x 100 -y 200 input.png output.nvsg
```

Encode options:
| Flag | Description | Default |
|------|-------------|---------|
| `-x` | X offset | 0 |
| `-y` | Y offset | 0 |
| `-u` | Anchor X (u) | 0 |
| `-v` | Anchor Y (v) | 0 |
| `--type` | Force NVSG type (0–4) | auto-detect |
| `--parts` | Number of parts, must be divisible by image height | 1 |

#### Image types

| Type | Name | Pixel format |
|------|------|-------------|
| 0 | Image | 24-bit BGR |
| 1 | Image32 | 32-bit BGRA |
| 2 | Parts | 32-bit BGRA, multiple frames |
| 3 | Mask | 8-bit grayscale |
| 4 | Gaiji | 8-bit grayscale (custom glyphs) |

### Bytecode

Disassemble an HCB bytecode file:
```
hime-tools hcb disasm input.hcb output/
```

Produces three files in the output directory:
- `{name}.asm` — disassembled instructions with labels
- `strings.txt` — extracted strings, one per line
- `config.yaml` — metadata (entry point, syscalls, game title, etc.)

Assemble back to HCB:
```
hime-tools hcb asm input/ output.hcb
```

Reads `{name}.asm`, `strings.txt`, and `config.yaml` from the input directory. Produces a byte-identical `.hcb` file.

##### Strings

The `strings.txt` file uses a simple flat text format:
- One string per line (content preserved exactly, no trimming)
- Blank lines are ignored
- Lines starting with `;` are comments
- `#include <file>` expands an included file (recursive, with cycle detection)
- `#reference <file>` can be used to emit metadata reference to another string file
- `#emit empty` emits an empty string
